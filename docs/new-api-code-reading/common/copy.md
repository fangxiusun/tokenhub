# copy.go 代码阅读文档

## 1. 全局总结

`copy.go` 提供了一个泛型深拷贝工具函数，基于 `github.com/jinzhu/copier` 库实现。该文件是项目中结构体深拷贝的统一入口，确保深拷贝操作具有一致的行为：深度复制所有字段，并忽略零值字段。

## 2. 依赖关系

| 依赖包 | 用途 |
|--------|------|
| `fmt` | 错误格式化 |
| `github.com/jinzhu/copier` | 结构体拷贝库，支持深度复制和选项配置 |

本文件无自定义类型定义。

## 4. 函数详解

### `DeepCopy[T any](src *T) (*T, error)`

- **功能**：对任意结构体进行深拷贝
- **类型参数**：`T any` — 任意类型
- **参数**：`src *T` — 源结构体指针
- **返回值**：
  - `*T` — 深拷贝后的新结构体指针
  - `error` — 拷贝过程中的错误
- **拷贝选项**：
  - `copier.Option{DeepCopy: true}` — 启用深度复制，递归复制嵌套结构体和切片
  - `copier.Option{IgnoreEmpty: true}` — 忽略零值字段，源中的零值不会覆盖目标中的已有值
- **错误处理**：
  - 源指针为 `nil` 时返回 `fmt.Errorf("copy source cannot be nil")`
  - `copier.CopyWithOption` 失败时直接返回错误

## 5. 关键逻辑分析

1. **泛型设计**：使用 Go 泛型（`[T any]`），避免了 `interface{}` 类型断言，提供编译时类型安全。

2. **深拷贝语义**：
   - `DeepCopy: true` 确保切片、映射、嵌套结构体等引用类型也被递归复制，而非浅拷贝引用
   - 这对于需要独立修改副本而不影响原始对象的场景至关重要

3. **忽略零值策略**：
   - `IgnoreEmpty: true` 意味着如果源字段是零值（`0`、`""`、`nil`、`false`），目标字段保持不变
   - 这在"部分更新"场景中特别有用：只拷贝有值的字段，保留目标的默认值

4. **nil 安全检查**：函数首先检查源指针是否为 `nil`，防止后续 `copier` 调用 panic。

5. **典型使用场景**：
   - 从数据库模型创建 DTO（数据传输对象）
   - 创建结构体的独立副本以进行修改
   - 从配置结构体创建运行时副本
