# i18n.go 代码阅读文档

## 1. 全局概述

本文件是后端国际化（i18n）系统的核心实现，基于 `nicksnyder/go-i18n/v2` 库。提供翻译初始化、语言检测、消息翻译等功能。支持中文简体、中文繁体和英文三种语言。

## 2. 依赖关系

- `embed` — Go 嵌入文件系统
- `strings` — 字符串操作
- `sync` — 同步原语
- `github.com/gin-gonic/gin` — Web 框架
- `github.com/nicksnyder/go-i18n/v2/i18n` — 国际化库
- `golang.org/x/text/language` — 语言标签处理
- `gopkg.in/yaml.v3` — YAML 解析
- `github.com/QuantumNous/new-api/common` — 通用工具
- `github.com/QuantumNous/new-api/constant` — 常量定义
- `github.com/QuantumNous/new-api/dto` — 数据传输对象

## 4. 函数详情

### Init

```go
func Init() error
```

初始化 i18n 系统：
1. 创建 i18n Bundle，设置默认语言为中文
2. 注册 YAML 反序列化函数
3. 从嵌入文件系统加载翻译文件（zh-CN.yaml, zh-TW.yaml, en.yaml）
4. 预创建各语言的 Localizer
5. 设置 `common.TranslateMessage` 函数

### GetLocalizer

```go
func GetLocalizer(lang string) *i18n.Localizer
```

获取指定语言的 Localizer。如果不存在则创建新的 Localizer（回退到默认语言）。

### T

```go
func T(c *gin.Context, key string, args ...map[string]any) string
```

从 Gin Context 中获取语言并翻译消息键。

### Translate

```go
func Translate(lang, key string, args ...map[string]any) string
```

翻译指定语言的消息键。如果翻译失败，返回消息键本身。

### GetLangFromContext

```go
func GetLangFromContext(c *gin.Context) string
```

从 Gin Context 中提取语言设置，按优先级检查：
1. 用户设置（`ContextKeyUserSetting`）— 如果已加载
2. 延迟加载用户语言（通过 `userLangLoaderFunc`）
3. 中间件设置的语言（`ContextKeyLanguage`）
4. Accept-Language 请求头
5. 默认语言（英文）

### ParseAcceptLanguage

```go
func ParseAcceptLanguage(header string) string
```

解析 Accept-Language 请求头，返回首选语言。

### normalizeLang

```go
func normalizeLang(lang string) string
```

标准化语言代码：
- `zh-tw*` → `zh-TW`
- `zh*` → `zh-CN`
- `en*` → `en`
- 其他 → 默认语言

### SupportedLanguages

```go
func SupportedLanguages() []string
```

返回支持的语言列表：`["zh-CN", "zh-TW", "en"]`

### IsSupported

```go
func IsSupported(lang string) bool
```

检查语言代码是否受支持。

### SetUserLangLoader

```go
func SetUserLangLoader(loader func(userId int) string)
```

设置用户语言加载函数（由 model 包调用以避免循环依赖）。

## 5. 关键逻辑分析

### 语言检测优先级

```
用户设置 > 延迟加载 > 中间件设置 > Accept-Language > 默认(英文)
```

这种多级回退机制确保在不同场景下都能获取正确的语言设置。

### 嵌入文件系统

使用 `//go:embed locales/*.yaml` 将翻译文件嵌入到二进制文件中，无需在部署时单独管理翻译文件。

### 线程安全

- 使用 `sync.Once` 确保初始化只执行一次
- 使用 `sync.RWMutex` 保护 Localizer 缓存的并发访问
- Localizer 采用双重检查锁定模式创建

### 延迟加载机制

`userLangLoaderFunc` 允许 model 包注册用户语言加载函数，避免循环依赖。在 session 认证场景下，用户设置可能未完全加载，此时通过用户 ID 从缓存/数据库延迟加载语言设置。

### common.TranslateMessage 集成

初始化时设置 `common.TranslateMessage = T`，使得 `common` 包中的工具函数也能使用 i18n 翻译。
