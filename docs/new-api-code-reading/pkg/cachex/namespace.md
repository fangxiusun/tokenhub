# namespace.go 代码阅读文档

## 1. 全局总结
该文件定义了 Namespace 类型，用于在混合缓存系统中隔离不同用例的键。通过为键添加命名空间前缀，避免不同缓存场景间的键冲突。

## 2. 依赖关系
- **strings**: 字符串处理（去除空格、冒号等）。

## 3. 类型定义
### Namespace
字符串类型，表示缓存键的命名空间（如 "channel_affinity:v1"）。

## 4. 函数详解
### prefix
内部方法，返回命名空间前缀：去除首尾空格和尾部冒号，添加冒号分隔符。空命名空间返回空字符串。

### FullKey
将原始键转换为带命名空间的完整键：去除首尾空格，若键已包含前缀则直接返回，否则添加前缀。

### MatchPattern
返回匹配命名空间下所有键的模式：空命名空间返回 "*"，否则返回 "prefix*"。

## 5. 关键逻辑分析
- **键隔离**: 通过命名空间前缀隔离不同用例，避免键冲突。
- **幂等性**: FullKey 方法确保重复添加前缀不会导致双重前缀。
- **模式匹配**: MatchPattern 用于 Redis SCAN 命令，高效扫描命名空间下的所有键。
