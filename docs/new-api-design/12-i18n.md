# 国际化层详细设计 (`i18n/`)

## 1. 概述
国际化层使用 `go-i18n` 库处理后端国际化。支持多种语言（英语、简体中文、繁体中文），提供统一的翻译接口。

## 2. 文件详细说明

### 2.1 `i18n.go` - 核心实现
**职责**: 核心 i18n 实现，带翻译加载和本地化器管理。

**关键函数**:
- **`Init()`**: 初始化 i18n 包，从嵌入式文件系统加载翻译文件。
  - 创建 `i18n.Bundle`，注册 YAML 反序列化函数
  - 加载嵌入式翻译文件：`locales/zh-CN.yaml`, `locales/zh-TW.yaml`, `locales/en.yaml`
  - 预创建支持语言的本地化器
  - 设置 `common.TranslateMessage = T` 函数
- **`GetLocalizer(lang string)`**: 返回指定语言的本地化器。
  - 支持语言标准化（如 `zh-tw` -> `zh-TW`）
  - 线程安全的本地化器缓存
  - 未知语言回退到默认语言（英语）
- **`T(c *gin.Context, key string, args ...map[string]any)`**: 使用 gin 上下文中的语言翻译消息键。
- **`Translate(lang, key string, args ...map[string]any)`**: 为指定语言翻译消息键。
- **`GetLangFromContext(c *gin.Context)`**: 从上下文提取语言设置。
  - 优先级：用户设置 > 延迟加载用户语言 > 中间件语言 > Accept-Language 头 > 默认语言
- **`ParseAcceptLanguage(header string)`**: 解析 Accept-Language 头。
- **`normalizeLang(lang string)`**: 将语言代码标准化为支持的格式。
- **`SupportedLanguages()`**: 返回支持的语言代码列表。
- **`IsSupported(lang string)`**: 检查语言代码是否受支持。

### 2.2 `keys.go` - 翻译键
**职责**: 包含应用程序中使用的所有翻译键。

### 2.3 `locales/` 目录
**职责**: 包含 YAML 格式的翻译文件。
- `zh-CN.yaml` - 简体中文翻译
- `zh-TW.yaml` - 繁体中文翻译
- `en.yaml` - 英语翻译

## 3. 语言检测优先级
1. **用户设置** (ContextKeyUserSetting) - 如果已加载
2. **延迟加载用户语言** - 使用用户 ID 从缓存/数据库加载
3. **中间件设置的语言** (ContextKeyLanguage) - 从 Accept-Language 头
4. **Accept-Language 头** - 直接解析（中间件未运行时的回退）
5. **默认语言** (英语)

## 4. 设计模式
- **单例模式**: `initOnce` 确保只初始化一次
- **缓存模式**: 本地化器缓存避免重复创建
- **延迟加载**: 用户语言支持延迟加载，避免不必要的数据库查询
- **回退机制**: 未找到翻译时返回键本身作为回退
