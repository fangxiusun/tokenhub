# 日志层详细设计 (`logger/`)

## 1. 概述
日志层提供应用程序的结构化日志功能，基于 Zap 日志库实现高性能、结构化的日志记录。

## 2. 文件详细说明

### 2.1 `logger.go` - 核心实现
**职责**: 核心日志器实现，带结构化日志支持。

**关键函数**:
- **`Init()`**: 初始化日志器，从环境变量加载配置。
  - 设置日志级别（debug, info, warn, error）
  - 配置输出目标（控制台、文件）
  - 启用日志轮转（按大小或时间）
- **`Info(msg string, fields ...zap.Field)`**: 记录信息级别日志。
- **`Error(msg string, fields ...zap.Field)`**: 记录错误级别日志。
- **`Debug(msg string, fields ...zap.Field)`**: 记录调试级别日志。
- **`Warn(msg string, fields ...zap.Field)`**: 记录警告级别日志。
- **`With(fields ...zap.Field)`**: 创建带额外字段的子日志器。
- **`Sync()`**: 刷新任何缓冲的日志条目。

## 3. 功能特性
- **结构化日志**: 使用 Zap 的强类型字段，避免字符串拼接
- **日志轮转**: 支持按文件大小和时间轮转，自动清理旧日志
- **多输出目标**: 同时输出到控制台和文件
- **日志级别配置**: 通过环境变量动态调整日志级别
- **请求上下文日志**: 在请求处理过程中记录上下文信息
- **性能优化**: Zap 的高性能设计，最小化日志记录的开销

## 4. 使用场景
- **请求日志**: 在 `middleware/logger.go` 中记录每个 HTTP 请求
- **错误日志**: 在控制器和服务层记录错误信息
- **审计日志**: 记录重要的业务操作（如用户登录、支付）
- **性能日志**: 记录慢查询和性能瓶颈
